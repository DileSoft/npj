Внешняя авторизация для НПЖ
--------
http://npj.ru/npjdev/modules/authorize_module
--------

Данный модуль позволяет авторизовать пользователей по внешней базе "логин-пароль".

Каждому логину сопоставляется аккаунт (учётная запись) и журнал в узле, который
указан в конфигурационном файле. Если такого журнала ещё не существует, то он
создаётся автоматически.

Модуль предоставляет архитектуру для реализации связок с произвольными хранилищами
и содержит три конкретных примера таких реализаций:

  * ConfigPrincipal -- авторизация по списку логинов-паролей, хранящихся в 
                       конфигурационном файле модуля

  * HackPrincipal -- авторизация посредством http-запроса к файлу, защищённому
                     стандартной http-авторизацией.

  * DbmsMysqlPrincipal и 
  * DbmsPostgresPrincipal -- авторизация по внешней БД (mysql/postgres), в которой есть таблица
                             с логинами и паролями (пароли могут храниться в шифрованном виде)

Для полноценной работы модуля необходимо устанавливать его на узле, работающим 
под управлением NPJ R1.9 или выше.

--------
Installation:
--------

1. РАЗМЕЩЕНИЕ ДИСТРИБУТИВА

Содержимое дистрибутива необходимо скопировать в ту же папку, где установлено
ПО НПЖ. 

Папка "authorize/" должна встать в один ряд с папками "core/", "lib/", "npj/".


2. ПОДКЛЮЧЕНИЕ МОДУЛЯ

Для подключения модуля необходимо разкомментировать следующую строку 
в файле config_modules.php:
  // include( "authorize/npj_config.php" );


3. НАСТРОЙКА МОДУЛЯ

После подключения необходимо отредактировать файл "authorize/npj_config.php"
так, чтобы он отражал те настройки, с которыми вы хотите использовать модуль.

При необходимости создания новой связки требуется реализация наследника
класса "NpjCustomPrincipalSuper". Реализация размещается в одноимённом классу файле, 
файл помещается в каталог "authorize/classes".
