NPJ Trako -- это программное обеспечение, являющееся "надстройкой" для НПЖ и 
предназначенное для поддержки управления рапортами об ошибках, предложениях 
и проблемах. По-английски это называется bug/issue/feature/incident tracker.

NPJ Trako работает как "модуль" ПО НПЖ, т.е. для того, чтобы использовать 
Трако, вам нужно установить себе и НПЖ. Или, наоборот, если вы уже используете 
ПО НПЖ, то вам достаточно доустановить себе этот небольшой модуль, чтобы 
получить работающий трекер.

Для полноценной работы NPJ Trako необходимо устанавливать его на узле, работающим 
под управлением NPJ R1.7 или выше.
  * http://npj.ru/npjdev/release -- текущий релиз НПЖ.


Если вы обнаружите какие-либо проблемы в работе трекера, пожалуйста,
опишите их в баг-трекере проекта: http://pixel-apes.com/npj-trako/trako

--------
Installation:
--------

1. РАЗМЕЩЕНИЕ ДИСТРИБУТИВА

Содержимое дистрибутива необходимо скопировать в ту же папку, где установлено
ПО НПЖ. Конфликта имён при переносе произойти не должно.

Папка "trako/" должна встать в один ряд с папками "core/", "lib/", "npj/".


2. ИНИЦИАЛИЗАЦИЯ БД

Для инициализации БД необходимо выполнить SQL-код "trako/sql/trako.sql",
заменив "{prefix}" на ваш префикс БД, например, на "r1_" или на "r17_"
или на то, что вы сможете найти в config_db.php в строке, начинающейся
с $this->db_prefix


3. ПОДКЛЮЧЕНИЕ МОДУЛЯ

Для подключения модуля необходимо разкомментировать следующую строку 
в файле config_modules.php:
  // include( "trako/npj_config.php" );


4. НАСТРОЙКА МОДУЛЯ (опционально)
Вы можете настроить поведение трекера, изменяя файл "trako/npj_config.php".

Documentation:
--------

Модуль будет работать после выполнения всех описанных операций по инсталляции.
Чтобы получить доступ к интерфейсу модуля, достаточно дописать "trako" в конце 
практически любого URL вашего узла.

Например: http://your-npj-node.ru/admin/trako -- трекер для аккаунта "admin"
